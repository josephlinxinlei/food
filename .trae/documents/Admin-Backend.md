# 开发者后台 - 食物贴纸管理

## 1. 功能概述

开发者后台用于管理饮食记录应用中的食物贴纸图片，支持：

* 上传新图片

* AI自动标准化处理（统一尺寸、背景、风格）

* 编辑贴纸信息（名称、分类、标签）

* 预览和调整

* 启用/禁用贴纸

## 3. 技术实现方案

### 3.1 图片处理流程

```
用户上传图片
    ↓
[1] 格式验证 (JPG/PNG/WEBP, < 10MB)
    ↓
[2] AI标准化处理
    ├─ 统一尺寸: 512x512px
    ├─ 背景处理: 去除背景/统一白色背景
    ├─ 风格化: 卡通化/扁平化
    └─ 色彩增强: 饱和度/对比度调整
    ↓
[3] 预览确认
    ↓
[4] 填写贴纸信息
    ↓
[5] 保存到数据库和CDN
```

### 3.2 API接口设计

```typescript
// 上传图片
POST /api/admin/upload
- 请求: FormData (file)
- 响应: { id, originalUrl, status }

// AI处理图片
POST /api/admin/process/:id
- 请求: { 
    resize: boolean,
    removeBg: boolean,
    cartoonify: boolean,
    enhance: boolean 
  }
- 响应: { processedUrl, previewUrl }

// 保存贴纸
POST /api/admin/stickers
- 请求: {
    imageUrl: string,
    name: string,
    category: string,
    tags: string[],
    status: 'active' | 'inactive'
  }
- 响应: { id, ... }

// 更新贴纸
PUT /api/admin/stickers/:id
- 请求: { name?, category?, tags?, status? }
- 响应: { success }

// 删除贴纸
DELETE /api/admin/stickers/:id
- 响应: { success }

// 获取贴纸列表
GET /api/admin/stickers
- 参数: { page, limit, category, search, status }
- 响应: { items, total, page, limit }

// 批量上传
POST /api/admin/batch-upload
- 请求: FormData (files[])
- 响应: { jobs: [{ id, status }] }
```

### 3.3 数据库模型

```sql
-- 食物贴纸表
CREATE TABLE food_stickers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name VARCHAR(100) NOT NULL,
  category VARCHAR(50) NOT NULL, -- breakfast, lunch, dinner, snack, fruit, drink, etc.
  tags TEXT[],
  original_url TEXT NOT NULL,
  processed_url TEXT NOT NULL,
  thumbnail_url TEXT,
  status VARCHAR(20) DEFAULT 'active', -- active, inactive, pending
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  created_by UUID REFERENCES auth.users(id)
);

-- 处理任务表（用于批量处理）
CREATE TABLE processing_jobs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  original_url TEXT NOT NULL,
  status VARCHAR(20) DEFAULT 'pending', -- pending, processing, completed, failed
  result_url TEXT,
  error_message TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  completed_at TIMESTAMP
);
```

### 3.4 AI处理服务

使用以下技术进行图片标准化：

1. **尺寸统一**: Sharp (Node.js) 或 ImageMagick
2. **背景去除**:

   * remove.bg API

   * 或本地模型: rembg (Python)
3. **卡通风格化**:

   * Stable Diffusion img2img

   * 或 CartoonGAN
4. **色彩增强**:

   * 自动调整饱和度/对比度

   * 统一色彩风格

## 4. 安全考虑

1. **认证授权**: JWT token，管理员角色验证
2. **文件验证**: 限制文件类型和大小
3. **图片审核**: AI审核上传内容，防止违规图片
4. **访问控制**: 后台仅对管理员开放
5. **操作日志**: 记录所有增删改操作

## 5. 部署配置

后台访问路径: `/admin`

* 独立路由，需要登录

* 与主应用共享认证状态

* 使用相同的数据库和存储服务
